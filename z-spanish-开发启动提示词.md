## 主提示词（粘贴这段）

我要开发一个自用的西班牙语背诵训练 App，叫 z-spanish。这是针对我个人使用特性（ADHD）设计的工具，不是商业产品——所有设计取舍以"是否真的会被我长期使用"为唯一标准，凡是增加操作摩擦、制造焦虑、或提供"看起来有用但会被用来逃避训练"的功能一律不做。

我附上了完整的设计规格文档（`z-spanish-原始需求.md` 和 `z-spanish-调整补充-v1.md`），以及一份可交互的 UI 原型（`z-spanish-prototype.html`）。请先完整读完再动手，不要开始就写代码。两份文档的关系：原始需求是地基，调整补充是在其之上的增量修正，**冲突时以调整补充文档为准**。

### 技术栈（已定，不要改）

- 后端：FastAPI + SQLite
- 前端：原生 HTML/CSS/JS，PWA（iPhone 主屏可添加）
- 部署：Docker Compose，目录 `/z/apps/z-spanish`，备份 `/z/backup/z-spanish`
- 数据必须存服务器端（我经常清浏览器数据，不能依赖 localStorage）
- PIN 由后端验证，不写在前端

### UI 参考

`z-spanish-prototype.html` 是已经和我确认过的高保真原型，涵盖 PIN 登录/首页/训练/基础库/统计五个页面的视觉与交互。请以它为 UI 基准还原，配色（主色深蓝 #0B1F3A）、布局、词块药丸样式、熟练度标签等都照原型走。

### 第一步只做这些，不要多做

1. **先跟我确认你对整个系统逻辑的理解**（内容准入规则、每日生成优先级、四档判定、间隔复习、熟练度阶段、静默计时），用你自己的话复述一遍，有歧义的地方问我，确认无误再动手。
2. **设计 SQLite 表结构**（词条表、复习状态表、每日记录表、时长记录表），先给我看 schema，我确认后再建。
3. **定义词条 JSON schema 并写好导入 + 校验逻辑**（校验规则见文档：动词要有人称块 + 搭配、名词要有冠词块 + 搭配句、介词 ≥2 条短语、example_es 不能为空）。

### 明确不要在第一版做的

见调整补充文档及原始需求"第一版不要做什么"清单：独立场景页、Bloques/Situaciones 一级页、Yo 大页面、Notas、Biblioteca 管理系统、Textos、Cloze、成长树、多训练模式、复杂熟练度面板、语音识别、倒计时、排行榜等一律不做。底部导航最多 4 个：首页 / 训练 / 基础库 / 统计。

### 关键约束提醒（容易被忽略，请特别注意）

- **熟练度阶段只在基础库显示，训练页绝对不显示**（会提前暴露答案，破坏主动回忆）
- **统计用"本月完成天数"不是"连续天数"**，不用百分比 / 进度条（避免全或无放弃模式与正确率焦虑）
- **near_correct 判定要宽松**：不区分大小写、无重音符号也算 near_correct 而非 wrong
- **静默计时**：训练页不显示实时秒表，只有一个"结束"按钮；忘记结束时以最后一次答题时间兜底，而不是关闭 App 的时间；时长只做日/周/月/年记录，不设目标、不比较
- **内容准确性最高优先级**——"宁少勿错"，错误数据会被间隔复习系统反复强化
- **自定义内容导入**：走 App 外生成 → 人工审核 → 基础库底部弱化入口批量导入，不做 App 内 AI 一键生成
- **每日新内容上限 5 个**是硬红线，即使状态好也不放宽；超出 20 题可"再来一组"但只从到期复习+需要加强抽取，不影响明天排期

请先做第一步的逻辑确认，不要急着写代码。

## 后续开发顺序建议（第一步确认通过后再往下）

1. 逻辑确认 + 数据库 schema + 词条 JSON schema 与导入校验（第一步）
2. 间隔复习引擎（2/4/7/14/30/60/90 天）+ 每日生成算法（优先级：到期复习 → 需要加强 → 混淆组 → 补新内容）
3. 四档判定逻辑（correct/near_correct/wrong/forgot）+ 熟练度阶段计算（初识/巩固中/稳固/长期记忆，只升级于按时答对，答错回退一级）
4. 后端 API + PIN 验证
5. 前端五页面（按原型还原）+ PWA 配置
6. 静默计时 + 统计页日/周/月/年聚合 + 复制给 GPT
7. Docker Compose 部署 + 每日备份脚本
8. 先灌入 Core 100 精准词条，跑通完整训练闭环，稳定使用一个月后再扩展 Core 300/500

## 内容建设并行进行

代码开发的同时，可以按调整补充文档第七节 + 第十四节的分类，在 App 外用 AI 逐步生成 Core 100 词条（严格遵守词块不孤立规则），人工审核后导入。先把 Core 100 做准，不要急着扩量。
