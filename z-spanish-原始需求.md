# z-spanish / Core Basics 原始需求文档

> 说明：这是项目最初的完整需求整理。后续的修正与增补见 `z-spanish-调整补充-v1.md`，两者冲突时以调整补充文档为准。

## 一句话定位

z-spanish 第一版不是综合西语学习平台，而是一个"西语生活基础背诵 + 主动输出训练器"。

核心只做一件事：中文意图 → 开口说西语 → 手动输入西语 → 系统检查 → 显示标准答案/例句/说明 → 错的进入需要加强 → 按间隔复习反复出现。

目标：把 A1-A2 生活西语底层表达背下来、说出来、写出来，形成自动反应。

## 一、为什么做这个项目

现在的问题不是完全没学过，而是：看懂≠会说，眼熟≠能输出，今天会≠明天还会，知道规则≠生活中能用出来。

所以这个项目不应该变成语法课、AI 课文系统、学习资料管理器。核心任务是：帮你把"生活中想说的话"固定成西班牙人自然会说的短句和语言块。

## 二、项目核心原则

1. **背诵是核心**：反复输出中文→西语，直到变成自动反应。
2. **主动输出优先**：不是选择题、不是看答案、不是点"我会"，而是自己想、自己说、自己输入、系统判断。
3. **基础库要全，但每天训练要少**：最终目标 Core 500，但每日只练 20 个（新内容最多5个，复习15个左右，旧内容优先）。
4. **中文是提示，不是直译**：中文提示是"真实意图"，不是逐字翻译。
5. **JSON 基础表必须准确**：错误数据会被系统反复强化，宁少勿错。先做 Core 100 精准版，再扩展。

## 三、第一版只保留 5 个核心页面

1. PIN 登录
2. 首页
3. 背诵训练
4. 基础库
5. 统计 / 复制给 GPT

## 四、第一版不要做什么

Bloques 独立一级页面、Situaciones 独立一级页面、Yo 大页面、Notas、Biblioteca 管理系统、Textos、Cloze、成长树、AI Organizador、多训练模式、学习时间统计、复杂熟练度面板、复杂语法课程、西语→中文、场景→西语、语音识别、倒计时、排行榜。

以后可以加，但第一版不做。核心是：每天打开就能背，背完就走。

## 五、页面设计

**PIN 登录页**：简单输入 PIN，后端验证，不写前端。

**首页**：只回答"今天我要背什么"。显示今天背诵训练/新内容5/复习15/总计20/需要加强3/[开始背诵]，加提示"先开口说，再输入西语"。

**背诵训练页**：一页只显示一道题。显示中文+类型标签+输入框+[检查][我不会]。不显示长解释，避免分心。

**答题反馈页**：答对显示"正确"+标准答案+例句+说明+[下一个]；答错显示"需要加强"+标准答案+你的答案+例句+说明+"请照着输入一遍"+[完成]。

**"我不会"流程**：不是失败，而是进入学习模式。显示标准答案→看例句→照着输入一遍→记录 forgot→进入需要加强→明天再出现。防止卡住退出。

**基础库页面**：不是复杂语法课，只是查看要背的内容。按分类展示，每条显示中文/西语/标签/例句/状态。

**统计 / 复制给 GPT**：只显示简单结果（今日完成/正确/接近正确/错误/没想起来/需要加强/连续学习），加[复制给GPT]。不要复杂图表，不要正确率焦虑。

## 六、基础库内容设计

最终目标 Core 500，分阶段：Core 100（最高频底层）、Core 300（生活常用扩展）、Core 500（A1-A2完整基础）。

## 七、必背分类（20类）

1. 方位、2. 指示词、3. 疑问词、4. 时间、5. 数字和数量、6. 否定、7. 连接词、8. 介词短语（介词不单独背，背短语）、9. ser/estar/hay、10. 高频动词人称块（按人称背，重点 yo/tú/él-ella-usted/nosotros；必背动词 ser/estar/tener/querer/poder/necesitar/saber/ir/hacer/decir/venir/dar/comprar/pagar/pedir/tomar/mirar/esperar/llamar/entender）、11. 代词、12. 冠词+高频名词（名词带冠词背）、13. 形容词、14. 比较和选择、15. 请求和能力、16. 礼貌表达、17. 生活固定句、18. 日常动作、19. 常用场景句（不是独立训练模式，只是基础库里的短句：酒吧/咖啡店、超市、问路、药店）、20. 拼写易混（está/esta、es/está、hay/tiene、aquí/allí、este/ese 等）。

## 八、每条 JSON 数据必须包含的字段

id、category、type_zh、type_es、subtype、zh、es、accepted_answers、example_es、example_zh、note、tags、confusion_group、is_active

示例：
```json
{
  "id": "loc_001",
  "category": "方位",
  "type_zh": "副词",
  "type_es": "adverbio",
  "subtype": "方位",
  "zh": "这里",
  "es": "aquí",
  "accepted_answers": ["aquí", "aqui"],
  "example_es": "Estoy aquí.",
  "example_zh": "我在这里。",
  "note": "aquí 表示\"这里\"，不要和 allí（那里）混淆。",
  "tags": ["A1", "必背", "高频"],
  "confusion_group": "aqui_alli",
  "is_active": 1
}
```

## 九、训练结果（4种）

- correct：完全正确
- near_correct：只差重音/标点/大小写
- wrong：意思或词错了
- forgot：点"我不会"

## 十、复习逻辑

核心原则：旧内容优先、新内容限量、错误回收、忘记回收、不要让用户自己判断熟不熟。

复习间隔：今天 / 第2天 / 第4天 / 第7天 / 第14天 / 第30天 / 第60天 / 第90天。

每日生成：先取到期复习 → 再取需要加强 → 再取混淆组 → 最后补新内容。旧内容超过20则今天不加新内容。

## 十一、UI 风格

简洁、大气、高端、手机优先、不幼稚、不花哨。主色深蓝 #0B1F3A。按钮深蓝底白字圆角12px高48px+。页面最大宽度480px，背景#F7F8FA，卡片白色圆角16-18px。

## 十二、部署架构

iPhone PWA → 域名 → Caddy/Nginx HTTPS → FastAPI → SQLite → 每日备份。前端原生 HTML/CSS/JS 或 Vue3，后端 FastAPI，数据库 SQLite，Docker Compose 部署，目录 /z/apps/z-spanish，备份 /z/backup/z-spanish。数据必须在服务器（经常清浏览器数据）。

## 十三、当前项目要怎么改

问题：已经变成综合学习平台。要收窄成 Core Basics 背诵输出训练器。

保留：FastAPI、SQLite、JSON 内容库、答案判定、间隔复习、错题订正、复制给 GPT。

隐藏或删除：Bloques/Situaciones 一级页面、Yo 大页面、Notas、Biblioteca、Textos、Cloze、成长树、多训练模式、学习时间统计、复杂熟练度展示。

## 十四、底部导航

最多 4 个：首页、训练、基础库、统计。不要 Hoy/Base/Bloques/Situaciones/Yo。

## 十五、复制给 GPT 的作用

不是让 GPT 看你练了多少，而是让 GPT 根据今天背过/错的/没想起来的内容，设计明天的生活化小课文，让已背和不稳的语言块在新课文里复现。

## 最终结论

每天背 20 个生活西语底层表达。中文提示，西语输出。错了回收，忘了再来，长期复习。核心是：Core 500 完整基础库 + 每日 20 个小剂量训练 + 中文→西语主动输出 + 准确 JSON + 间隔复习 + 错题回收 + 复制给 GPT 生成复现小课文。
